# 月の満ち欠け・日食月食シミュレーター AI 指示書

## 0. 共通事項

本仕様書は、既存の参考プロジェクト群と同じ設計思想・運用前提で動作する Web 教材の作成指示書である。個別機能に入る前に、全テーマ共通の前提を以下にまとめる。以降の章は、この共通事項を前提として本テーマ固有の要件のみを記述する。

### 0.1 参考プロジェクト

- [GEO-LEARNER](https://takayuki-minagawa.github.io/GEO-LEARNER/) — 観測視点切替 + 学習モジュールの三層構成
- [weather-front-sim](https://takayuki-minagawa.github.io/weather-front-sim/) — 時系列操作 + 観測値グラフ + 学習パネル + クイズ
- [web-planetarium](https://takayuki-minagawa.github.io/web-planetarium/) — 地点・時刻に応じた星空描画と i18n / テーマ切替

これらと同じ設計思想（操作パネル＋可視化ビュー＋学習パネル＋クイズ、i18n、ライト／ダーク切替、静的配信）を踏襲する。

### 0.2 共通仕様

- 静的フロントエンド構成とする（React + TypeScript + Vite を基本、軽量なら Vanilla JS + HTML + CSS も可）
- GitHub Actions により GitHub Pages へデプロイする
- 日本語 UI を基本とし、英語切替（i18n）に対応する
- ライト／ダークテーマ切替に対応する
- PC・タブレット・スマホ対応のレスポンシブ UI にする
- 画面は「操作パネル」「可視化ビュー」「学習パネル（解説＋クイズ）」の三層構成を基本とする
- 教育用途のコンテンツであり、実測・予報・診断など厳密な数値提供を目的にしない
- 公開リポジトリ前提の表現で README と仕様書を書き、個人情報・認証情報・秘密鍵は含めない

### 0.3 共通で再利用したいコンポーネント

- タイムスライダー、再生・一時停止・速度変更コントロール
- 学習パネル・クイズフレーム（状態に応じた解説文生成と正誤判定）
- i18n ロード機構、ライト／ダークテーマ切替
- 3D シーン基盤（Three.js + React Three Fiber）: 3D 系テーマで共用
- 2D グラフ・波形描画基盤（Canvas / D3.js）: 時系列グラフ系テーマで共用
- SVG 編集系 UI（図形選択・ドラッグ・履歴）: 図形操作系テーマで共用

### 0.4 学習パネルの共通方針

- 現在の状態・入力に対する解説を自動生成する
- 観察の着目点や誘導テキストを出す
- クイズモードを併設し、主要な誤解ポイントを問題化する
- 言語切替に追従して文言が再生成される構造にする

## 1. 目的

ブラウザ上で動作する Web アプリとして、太陽・地球・月の位置関係を操作しながら、月の満ち欠けと日食・月食の発生条件を同時に視覚的に理解できる教材を作成する。

高校地学および地学基礎の定番単元であり、平面図と実際の天体配置の対応関係で誤解が生じやすい分野であるため、次の点を直感的に掴めることを目指す。

- 月の公転位置と地球から見た月の形（月相）の対応
- 日食・月食が新月・満月のときだけ起き、毎月起きるわけではない理由（月の軌道傾き）
- 本影・半影と、皆既食・部分食・金環食の違い
- 地球から見た月の見え方と、太陽系を俯瞰したときの配置の二重視点

## 2. 開発方針

### 2.1 基本方針

- フロントエンドのみで完結する静的 Web アプリとする
- GitHub Pages で公開可能な構成にする
- PC・タブレット・スマホでの利用を想定したレスポンシブ UI
- 日本語 UI を基本とし、英語切替を用意する
- ライト／ダークテーマ切替に対応

### 2.2 推奨技術スタック

- フレームワーク: React + TypeScript + Vite
- 3D 描画: Three.js + React Three Fiber + drei
- スタイリング: CSS Modules または Tailwind CSS
- 状態管理: React Hooks（Zustand があれば尚可）
- デプロイ: GitHub Actions + GitHub Pages

### 2.3 画面構成の基本

メイン領域を2つ並列表示するか、タブで切り替える構成を推奨する。

- **俯瞰ビュー**: 太陽・地球・月の配置を 3D で俯瞰表示
- **地球視点ビュー**: 地球から見た月の姿（満ち欠け）を拡大表示

---

## 3. 必須機能

### 3.1 月の公転操作

- 月の公転角度をスライダーまたはドラッグで 0°〜360° 操作可能にする
- 公転に追従して、俯瞰ビューで月・地球・太陽の位置関係が更新される
- 地球視点ビューの月相（新月・上弦・満月・下弦など）がリアルタイムに変化する
- 公転周期に従った自動再生モード（再生／一時停止／速度変更）

### 3.2 月相の可視化

- 地球視点ビューで月を拡大表示し、太陽光による明暗を正確に描画
- 現在の月齢（例: 14.8 日）と月相名（新月・三日月・上弦・十三夜・満月・十六夜・下弦・二十六夜 など）を表示
- 位相角（太陽 − 地球 − 月 のなす角）を数値で表示

### 3.3 日食・月食の判定

- 月の軌道に白道傾斜（約 5.14°）を取り入れる
- 新月時に太陽・月・地球がほぼ一直線に並ぶと日食
- 満月時に太陽・地球・月がほぼ一直線に並ぶと月食
- 俯瞰ビューで本影・半影の円錐を半透明で描画
- 食の種類（皆既／部分／金環／半影月食）を判定して表示
- 条件を満たしたときに「今、日食が起きています」などの学習テキストを表示

### 3.4 操作パネル

- 時間スライダー（1 日単位と時間単位の粗密切替）
- 月の軌道傾斜角 ON/OFF（OFF にすると毎月食になることを体験できる）
- 月の位置のクイック移動ボタン（新月／上弦／満月／下弦）
- 日食が起きる位置／月食が起きる位置への一発移動ボタン
- 俯瞰ビューのズーム、回転、リセット

### 3.5 学習パネル

- 現在の状態に対する解説テキスト（月相、配置、食の有無）
- 「なぜ毎月食が起きないのか」など学習の着眼点を提示
- クイズモード
  - 月相名クイズ
  - 配置図から月相を答える問題
  - 食が起きる条件の正誤問題

## 4. あると良い機能

- 実日付入力で、その日の月相と食の有無を近似表示
- 日食・月食の年間一覧からシミュレーションへジャンプ
- 本影・半影を通過する速度のアニメーション
- 地球の自転を加えた「どこから見えるか」マップ
- 子ども向けの簡易モード（用語をやさしく）
- URL クエリで月位置・日時を共有
- スクリーンショット保存
- PWA 対応

## 5. データ・計算設計

### 5.1 座標モデル

- 太陽を原点に固定
- 地球は円軌道で近似（必要な精度を超えた楕円軌道は使わない）
- 月は地球周りを円軌道で近似し、白道傾斜角のみ再現
- スケールは視認性のために圧縮して描画してよい（実スケールとは別に描画用スケールを持つ）

### 5.2 計算項目

- 月の位相角と照度率
- 太陽 − 月の見かけ方向
- 日食条件: 太陽方向と月方向の角距離が月の視半径＋太陽の視半径以下
- 月食条件: 地球影（本影・半影）内に月が入るか
- 月相名の閾値判定

### 5.3 学習テキストデータ

`learningTexts.ts` として状態ごとの解説文を切り出し、将来的な多言語化に備える。

## 6. アニメーションとビジュアル

- 俯瞰ビューでは月の軌跡を薄いラインで残像表示
- 新月付近では月と太陽の視黄経差を強調する補助線を表示
- 食のとき、本影に入った領域を赤系でハイライト
- 地球視点ビューの背景は、時間に応じて空色〜夜空グラデーション
- 月相変化は 60 fps を目安になめらかに補間
